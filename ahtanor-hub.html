The Structural Principle

How This Archive Is Organized

Every page belongs to one of three Spheres — the Paracelsian Tria Prima: Corpus (body, material world, craft), Anima (soul, psyche, culture), and Spiritus (spirit, abstraction, cosmology). Within each Sphere live three Domains, drawn from the Pythagorean fourfold descent from unity to matter. Every domain can hold an unlimited number of pages. New pages are simply added to the correct domain — the architecture holds itself.

The Pythagoreans taught that all of existence descends through four levels: the Point (unity, source), the Line (duality, relationship), the Plane (third dimension, expression), and the Solid (the world, material form). Your pages live at the level of the Solid — concrete artifacts of lived experience — but each one contains within it the higher principles of its domain.

The Operator's Manual

How to Add a New Page to the Archive

The architecture is designed for continuous expansion. Every new page follows the same four-step ritual.

I
Determine the Sphere and Domain

Ask: Is this primarily material/craft (Corpus), soul/cultural (Anima), or philosophical/structural (Spiritus)? Then: which of the three domains within that sphere does it belong to? If it genuinely crosses all three, it goes in Quinta Essentia.

II
Name and place the HTML file

Save your new page into the same folder as this index.html. Use a clear, lowercase, hyphenated filename. Example:

your-site/ ├── index.html ← this hub ├── the-unbecoming.html ← Anima / Interior ├── music-production.html← Corpus / Workshop ├── france-plan.html ← Corpus / Territory ├── the-return.html ← Anima / Interior └── herbalism-primer.html← Corpus / Vehicle (new)
III
Add an entry row to the correct domain

Find the correct domain column in this file. Replace a placeholder entry (or add after the last real entry) with this HTML block:

<a class="entry" href="your-filename.html"> <div class="entry-dot"></div> <div class="entry-content"> <div class="entry-title">Your Page Title</div> <div class="entry-note">One sentence description</div> <div class="entry-date">Year</div> </div> </a>
IV
Update tag counts (optional but recommended)

In the Tag Vocabulary section, increment the count next to each relevant tag. This keeps the cross-reference system accurate as the archive grows. When a tag accumulates 5+ entries, consider whether it deserves its own sub-domain.

Going Live · Your Domain · Your Archive

Deployment in Two Paths

You have a registered domain. These are the two clearest paths to getting this live, in order of simplicity.

★ Recommended · Simplest
Netlify Drop
Drag, drop, live in 60 seconds. Then point your domain.
  • Go to netlify.com → create a free account
  • From the dashboard, drag your entire project folder onto the page — it deploys instantly to a temp URL
  • Go to Site Configuration → Domain Management → Add custom domain → enter your domain name
  • Netlify gives you two nameserver addresses (e.g. dns1.p01.nsone.net)
  • Log into your domain registrar (GoDaddy, Namecheap, etc.) → find DNS/Nameserver settings → replace existing nameservers with Netlify's two
  • Wait 15 min – 48 hours for propagation. Usually under an hour.
  • SSL/HTTPS is automatic — Netlify handles it with Let's Encrypt
  • To update: just re-drag the updated folder, or connect a GitHub repo for automatic deploys
Advanced · Auto-Deploy
GitHub + Netlify
Push a file, the site updates automatically. No manual re-uploading ever.
  • Create a free GitHub account → New Repository → name it (e.g. "the-athanor")
  • Upload your HTML files to the repository (GitHub has a drag-and-drop UI — no terminal needed)
  • In Netlify: Add New Site → Import from Git → GitHub → select your repo
  • Netlify auto-detects it's a static site. Hit Deploy. Done.
  • Point your domain as above (nameservers to Netlify)
  • To add a new page: drag the new HTML file into your GitHub repo → Netlify detects the change and redeploys in ~30 seconds
  • This is the long-term architecture — every new page you make with Claude gets added to the repo, and the live site updates automatically
